# Myceliate CLI v1.1 — Daily Driver Sprint Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the five UX gaps surfaced by the v1 live-smoke run so Myceliate behaves like a daily-driver coding assistant (Claude Code-style), not a single-shot demo. Add a multi-turn REPL loop with `.env` auto-load, surface tool calls and their lifecycle in the TUI, wire the existing bash tool path through HITL approval and BullMQ, and render tool results inline.

**Architecture:** Four phases mirroring v1's cadence. Phase 12 wraps the existing `runReactLoop` in an outer prompt loop with engine continuity; the orchestrator gains a new `tool_result` `StreamEvent` (Phase 13) so the TUI can show explicit lifecycle transitions for each tool call (`running → completed | failed | rejected`); Phase 14 finally integrates the Phase 5 BullMQ worker, Phase 7 HITL gate, and Phase 4 tool registry into a real bash tool the agent can invoke with approval; Phase 15 polishes the result-rendering surface so tool outputs render inline as collapsible blocks.

**Tech Stack:** Existing v1 stack — Node.js ≥20.11, TypeScript strict, Ink + React 18, BullMQ + ioredis, Zod, Vitest. New runtime dependency: `dotenv` (or equivalent — locked to `dotenv` for ergonomics; see Phase 12 Task 81).

**Scope check:** Closing the gaps surfaced by the live-smoke run on 2026-05-03. Deferred (do NOT implement here): session resume by id, slash commands beyond `/quit`, JIT skill auto-load, subagent spawning (R8/R9), tool result diff rendering for `write_file`, syntax highlighting in tool result blocks, V4 `ThinkingOptions` wire-shape fix (carry-over from live smoke; tracked separately as F7). See **Deferred to v1.2** at the bottom of this doc.

**v1.0 fast-follow promoted into this sprint:** Local-history redaction (Task 81a). The v1.0 ultrareview documented that `.myceliate/history/<session>.md` writes raw `Message` content while egress goes through `redactSecrets`. // src/memory/conversationLog.ts — replace the existing renderTurn function
import { redactJsonLeaves, redactSecrets } from '../security/redactor.js';

function renderTurn(m: Message): string {
  switch (m.role) {
    case 'system':
    case 'user':
    case 'assistant': {
      // Task 81a: redact every cleartext channel before disk write so
      // `.myceliate/history/<session>.md` carries the same R11 guarantees as
      // egress payloads. Tool-call args are walked per-leaf (redactJsonLeaves)
      // because the env_value pattern's greedy `\S+` would otherwise consume
      // the closing `}` of the JSON envelope when applied to the assembled
      // string — matching F1's adapter-level treatment.
      const content = m.content ? redactSecrets(m.content) : '';
      const reasoning =
        m.role === 'assistant' && m.reasoning_content
          ? `\n<details><summary>reasoning</summary>\n\n${redactSecrets(m.reasoning_content)}\n\n</details>\n`
          : '';
      const tools =
        m.role === 'assistant' && m.tool_calls?.length
          ? `\n\n**tool_calls:** ${m.tool_calls
              .map((tc) => `${tc.name}(${JSON.stringify(redactJsonLeaves(tc.args))})`)
              .join(', ')}\n`
          : '';
      return `\n\n### ${m.role}\n\n${content}${reasoning}${tools}`;
    }
    case 'tool':
      return `\n\n### tool (${m.result.tool_use_id}) ${m.result.is_error ? 'ERROR' : 'OK'}\n\n\`\`\`\n${redactSecrets(m.result.content)}\n\`\`\``;
  }
}
```

// src/ui/PromptInput.tsx
import { Box, Text, useInput } from 'ink';
import { useState } from 'react';
import type React from 'react';

/**
 * Inline Ink prompt input for the REPL loop. Renders a single-line buffer
 * with a green chevron prompt. Enter submits and clears; backspace edits.
 *
 * U3-compliant: no Clack mid-Ink-render. The agent loop owns input via Ink
 * for the entire session (Clack only runs pre-Ink during onboarding).
 */
export function PromptInput({
  onSubmit,
}: { onSubmit: (text: string) => void }): React.JSX.Element {
  const [buffer, setBuffer] = useState('');

  useInput((input, key) => {
    if (key.return) {
      // Submit on Enter; clear the buffer for the next turn.
      const submitted = buffer;
      setBuffer('');
      onSubmit(submitted);
      return;
    }
    if (key.backspace || key.delete) {
      setBuffer((b) => b.slice(0, -1));
      return;
    }
    if (key.tab) return; // Tab is reserved for ReasoningBlock toggle.
    if (input.length > 0 && !key.ctrl && !key.meta) {
      setBuffer((b) => b + input);
    }
  });

  return (
    <Box marginTop={1}>
      <Text color="green">{'> '}</Text>
      <Text>{buffer}</Text>
      <Text color="gray">{'▎'}</Text>
    </Box>
  );
}
```

// src/ui/App.tsx — replace existing AppState + return
import { PromptInput } from './PromptInput.js';

export type CompletedTurn = {
  userInput: string;
  content: string;
};

export type AppState = {
  userInput: string;
  reasoning: ReasoningState | null;
  content: string;
  approvalRequest: ApprovalRequest | null;
  /** REPL phase: streaming = a turn is in flight; awaiting_input = ready for next prompt. */
  phase: 'streaming' | 'awaiting_input';
  /** Append-only log of completed turns (rendered above the live region). */
  turns: CompletedTurn[];
};

export function App({
  state,
  onApprovalResponse,
  onPromptSubmit,
}: {
  state: AppState;
  onApprovalResponse?: (r: ApprovalResponse) => void;
  onPromptSubmit?: (text: string) => void;
}): React.JSX.Element {
  const [reasoningExpanded, setReasoningExpanded] = useState(false);
  useInput((_input, key) => {
    if (key.tab) setReasoningExpanded((prev) => !prev);
  });

  return (
    <Box flexDirection="column" paddingX={1}>
      {state.turns.map((t, i) => (
        // biome-ignore lint/suspicious/noArrayIndexKey: append-only log, indices are stable.
        <Box key={i} flexDirection="column" marginBottom={1}>
          <Box>
            <Text color="green">{'> '}</Text>
            <Text>{t.userInput}</Text>
          </Box>
          <Text>{t.content}</Text>
        </Box>
      ))}
      {state.phase === 'streaming' && (
        <>
          <Box marginBottom={1}>
            <Text color="green">{'> '}</Text>
            <Text>{state.userInput}</Text>
          </Box>
          {state.reasoning !== null && (
            <ReasoningBlock
              text={state.reasoning.text}
              phase={state.reasoning.phase}
              durationMs={
                (state.reasoning.endedAtMs ?? Date.now()) - state.reasoning.startedAtMs
              }
              expanded={reasoningExpanded}
            />
          )}
          {state.content.length > 0 && <ContentStream text={state.content} />}
        </>
      )}
      {state.approvalRequest !== null && (
        <ApprovalPrompt
          request={state.approvalRequest}
          onResponse={onApprovalResponse ?? (() => {})}
        />
      )}
      {state.phase === 'awaiting_input' && (
        <PromptInput onSubmit={onPromptSubmit ?? (() => {})} />
      )}
    </Box>
  );
}
```

// src/runtime/replSession.ts
import type { DeepSeekClient } from '../adapters/DeepSeekClient.js';
import type { Message } from '../adapters/messages.js';
import type { StreamEvent } from '../adapters/streamEvent.js';
import { QueryEngine } from '../orchestrator/QueryEngine.js';
import { runReactLoop } from '../orchestrator/reactLoop.js';
import type { ToolRegistry } from '../tools/registry.js';

export type ReplSessionOptions = {
  client: DeepSeekClient;
  tools: ToolRegistry;
  model: string;
  cwd: string;
  systemPrompt?: string;
  workingBudget?: number;
  /** Streamed events from each turn — consumer wires this to App state. */
  onState: (ev: StreamEvent) => void;
  /** Fires once after each terminal turn with the engine snapshot. */
  onTurnComplete: (history: readonly Message[]) => void;
  /** Resolves with the next user prompt (or "/quit" / empty to exit). */
  readNextPrompt: () => Promise<string>;
};

const QUIT_TOKENS = new Set(['/quit', '/exit', '']);

export async function runReplSession(opts: ReplSessionOptions): Promise<void> {
  const engine = new QueryEngine({
    systemPrompt: opts.systemPrompt ?? 'You are myceliate, an autonomous CLI agent.',
    workingBudget: opts.workingBudget ?? 200_000,
  });

  while (true) {
    const prompt = (await opts.readNextPrompt()).trim();
    if (QUIT_TOKENS.has(prompt)) return;

    engine.appendUser(prompt);
    for await (const ev of runReactLoop({
      client: opts.client,
      engine,
      tools: opts.tools,
      model: opts.model,
      cwd: opts.cwd,
    })) {
      opts.onState(ev);
    }
    opts.onTurnComplete(engine.snapshot());
  }
}

      promptResolver = null;
      r(text);

          });
        } else if (ev.type === 'content_delta') {

            rerender({
              ...state,
              reasoning: { ...state.reasoning, phase: 'complete', endedAtMs: Date.now() },
            });

          contentText += ev.text;
          rerender({ ...state, content: contentText });

          reasoningText = '';
          contentText = '';
          reasonStartedAt = Date.now();

        } else if (ev.type === 'error') {
          logger.error({
            event: 'stream_error',
            message: ev.cause instanceof Error ? ev.cause.message : String(ev.cause),
          });
        }
      },
      onTurnComplete: async (snapshot) => {
        // Flush new messages to the conversation log. Skip messages already written:
        // first turn's user message is written below before the loop reads its first prompt.
        const fromIndex = state.turns.length === 0 && firstPromptConsumed ? 1 : 0;
        for (const m of snapshot.slice(fromIndex)) await conversation.appendTurn(m);
        const newTurn: CompletedTurn = { userInput: state.userInput, content: contentText };
        const turns = [...state.turns, newTurn];
        // Reset live region; show prompt input for next turn.
        reasoningText = '';
        contentText = '';
        rerender({
          userInput: '',
          reasoning: null,
          content: '',
          approvalRequest: null,
          phase: 'awaiting_input',
          turns,
        });
      },
      readNextPrompt: async () => {
        if (!firstPromptConsumed) {
          firstPromptConsumed = true;
          await conversation.appendTurn({ role: 'user', content: onboarding.initialPrompt });
          return onboarding.initialPrompt;
        }
        return new Promise<string>((resolve) => {
          promptResolver = resolve;
        }).then((text) => {
          rerender({
            userInput: text,
            reasoning: null,
            content: '',
            approvalRequest: null,
            phase: 'streaming',
            turns: state.turns,
          });
          return text;
        });
      },
    });
  } finally {
    await logger.flush();
    ink.unmount();
  }
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});

// src/tools/bash.ts
import { randomUUID } from 'node:crypto';
import type { Queue, QueueEvents } from 'bullmq';
import { z } from 'zod';
import type { HitlGate } from '../security/hitlGate.js';
import type { BashJobData, BashJobReturn } from '../queue/queues.js';
import type { Tool } from './registry.js';

export type BashToolDeps = {
  hitl: HitlGate;
  queue: Queue<BashJobData, BashJobReturn>;
  queueEvents: QueueEvents;
  defaultTimeoutMs?: number;
};

const BashInput = z.object({
  command: z.string().min(1),
  /** Optional override; defaults to the orchestrator's cwd. */
  cwd: z.string().optional(),
  /** Optional override; defaults to deps.defaultTimeoutMs. */
  timeoutMs: z.number().int().positive().optional(),
});
type BashInput = z.infer<typeof BashInput>;

function formatResult(r: BashJobReturn): string {
  return [
    `exitCode: ${r.exitCode}`,
    r.timedOut ? 'timedOut: true' : null,
    r.truncated ? 'truncated: true' : null,
    r.stdout ? `stdout:\n${r.stdout}` : null,
    r.stderr ? `stderr:\n${r.stderr}` : null,
  ]
    .filter(Boolean)
    .join('\n');
}

export function createBashTool(deps: BashToolDeps): Tool<BashInput> {
  const defaultTimeout = deps.defaultTimeoutMs ?? 30_000;
  return {
    name: 'bash',
    description:
      'Execute a shell command in the agent cwd. Returns exit code, stdout, stderr, and truncation/timeout flags. Dangerous patterns require HITL approval.',
    capability: 'execution',
    inputSchema: BashInput,
    run: async (input, ctx) => {
      const cwd = input.cwd ?? ctx.cwd;
      const verdict = await deps.hitl.checkBash({ command: input.command, cwd });
      if (!verdict.allowed) {
        throw new Error(`HITL rejected: ${verdict.feedback}`);
      }

      const toolUseId = randomUUID();
      const job = await deps.queue.add(
        'bash',
        {
          command: input.command,
          cwd,
          timeoutMs: input.timeoutMs ?? defaultTimeout,
          toolUseId,
        },
        { jobId: toolUseId },
      );
      const result = await job.waitUntilFinished(deps.queueEvents);
      return formatResult(result);
    },
  };
}
```

// src/runtime/workerLifecycle.ts
import { spawn, type ChildProcess } from 'node:child_process';

export type WorkerHandle = {
  child: ChildProcess;
  shutdown: () => Promise<void>;
};

/**
 * Spawn `pnpm queue:worker` as a detached subprocess so the bash tool's
 * BullMQ jobs have a consumer. Returns a handle exposing the child and
 * an idempotent `shutdown` that SIGTERM-then-SIGKILLs.
 *
 * Caller is responsible for calling `shutdown()` in the main `try/finally`
 * block alongside `ink.unmount()` and `logger.flush()`.
 */
export function startWorker(): WorkerHandle {
  const child = spawn('pnpm', ['queue:worker'], {
    stdio: ['ignore', 'pipe', 'pipe'],
    detached: false,
  });
  let shuttingDown = false;
  const shutdown = async (): Promise<void> => {
    if (shuttingDown) return;
    shuttingDown = true;
    if (child.exitCode !== null) return; // already exited
    child.kill('SIGTERM');
    await new Promise<void>((resolve) => {
      const timer = setTimeout(() => {
        child.kill('SIGKILL');
        resolve();
      }, 2000);
      child.once('exit', () => {
        clearTimeout(timer);
        resolve();
      });
    });
  };
  return { child, shutdown };
}
```

export function ToolCallCard({
  card,
  expanded = false,
}: { card: ToolCallCardState; expanded?: boolean }): React.JSX.Element {
  const previewLines = card.preview ? card.preview.split('\n') : [];
  const COLLAPSED_LINES = 5;
  const visibleLines = expanded ? previewLines : previewLines.slice(0, COLLAPSED_LINES);
  const hiddenCount = previewLines.length - visibleLines.length;

  return (
    <Box flexDirection="column" marginY={1} borderStyle="round" paddingX={1}>
      {/* … existing header line … */}
      {card.status === 'completed' && previewLines.length > 0 && (
        <Box flexDirection="column" marginTop={1}>
          {visibleLines.map((line, i) => (
            // biome-ignore lint/suspicious/noArrayIndexKey: stable lines.

          ))}
          {hiddenCount > 0 && <Text color="gray">{`… ${hiddenCount} more lines`}</Text>}
        </Box>
      )}
      {/* … existing failed/rejected branches … */}
    </Box>
  );
}

const REDACT_RE = /\[REDACTED:[^\]]+\]/g;

function HighlightedLine({ line }: { line: string }): React.JSX.Element {
  // Split on the redaction marker to colour-segment the line.
  const parts: Array<{ text: string; redacted: boolean }> = [];
  let lastIndex = 0;
  for (const m of line.matchAll(REDACT_RE)) {
    if (m.index === undefined) continue;
    if (m.index > lastIndex) parts.push({ text: line.slice(lastIndex, m.index), redacted: false });
    parts.push({ text: m[0], redacted: true });
    lastIndex = m.index + m[0].length;
  }
  if (lastIndex < line.length) parts.push({ text: line.slice(lastIndex), redacted: false });
  return (
    <Text>
      {parts.map((p, i) => (
        // biome-ignore lint/suspicious/noArrayIndexKey: stable segments.
        <Text key={i} color={p.redacted ? 'magenta' : 'gray'}>
          {p.text}
        </Text>
      ))}
    </Text>
  );
}
```

## Lessons from v1 carryforward

These rules emerged from the v1 cadence and apply unchanged here:

1. **Per-phase `feat/phase-N` branches with `--no-ff` summary merges.** No worktrees — Patrick prefers the trunk-and-feature-branch rhythm with single-line summary commits.
2. **Fresh subagent per phase + two-stage review** (spec-conformance + code-quality). Apply MAJOR/MINOR fixes inline; document deferred items in memory under a `Phase N review follow-ups deferred to v2:` heading.
3. **Plan-defect fixes during implementation are recorded in memory as `Phase N plan-defect fixes during implementation:`** so future me knows what diverged from this doc and why.
4. **Conversational clarification preferred over AskUserQuestion** — when in doubt, just ask Patrick in plain text.
5. **Lesson transfer between phases** — Phase 8 hardenings carried into Phase 9 carried into Phase 10, etc. Don't re-derive the same defensive patterns; reach for the ones already proven (e.g., null-guard for `m.content`, idempotence guards on transforms, try/finally around the entire interactive surface).
6. **Memory updates are part of each phase merge** — `MEMORY.md` index and `project_myceliate.md` entry both get updated at merge time, not after the fact.
