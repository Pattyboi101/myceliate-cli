# Myceliate v1.3 — Spores: Sector-Aware Persona Bundles

**Status:** design / pre-implementation
**Baseline:** `main` @ `52fc967` (Phase 18 merge, v1.2 tag)
**Successor of:** [v1.2 hardening sprint](../plans/v1.2-hardening-sprint.md)
**Implementation plan:** to be written via `superpowers:writing-plans` after this spec is approved

---

## 1. Mission

v1.3 ships **Spores** — domain-scoped bundles of persona sub-agents that the orchestrator germinates when user intent matches a sector. A spore is the operationalisation of "you're trying to run a business — let me run these particular agents".

This is Myceliate's first feature that distinguishes it from coding-only CLI agents (Claude Code, OpenAI Codex CLI, OpenCode, Crush, Aider, Goose, Plandex). Pre-design research showed every published skill catalogue is ~80% engineering; the "non-coding sector pack" space is uncontested.

## 2. Vision Brief

Where v1.0 shipped a usable CLI agent and v1.1/v1.2 hardened it (resume, redaction, hermetic env, concurrent HITL), **v1.3 expands what the agent CAN BE**. Instead of one generalist orchestrator, the user gets a network of personas that activate based on context:

- Run myceliate in your IndieStack project → orchestrator detects "running a business" → germinates `solo-business` spore → CEO / outreach / marketing / pricing-analyst / etc become spawnable
- Run myceliate in your dissertation project → germinates `research` spore → lit-reviewer / methodologist / data-extractor / writer / etc become spawnable
- Run myceliate anywhere with no clear sector → bare orchestrator, opt-in via `/spore pin <name>`

Each spore is an environment of capabilities you germinate into the running orchestrator, just as a mycelium network grows into different substrates. The mycology metaphor stays load-bearing throughout the feature: catalog · germinate · pin · unpin.

## 4. Architecture

### 4.1 Spore Anatomy

A spore is a directory under one of the three tiers:

```
solo-business/
├── SKILL.md                    # Sector overview (frontmatter + body)
├── myceliate.yaml              # Myceliate-specific manifest
└── agents/
    ├── ceo/SKILL.md            # Persona instructions
    ├── outreach/SKILL.md
    ├── marketing/SKILL.md
    ├── copywriter/SKILL.md
    ├── pricing-analyst/SKILL.md
    ├── ops-manager/SKILL.md
    └── customer-success/SKILL.md
```

The directory layout is **portable to Claude Code and OpenAI Codex CLI** if the user copies it there — both consume `SKILL.md` files; only the `myceliate.yaml` and the `spawn_subagent` orchestration are Myceliate-specific.

### 4.2 `myceliate.yaml` Manifest

Located at sector root. Myceliate-specific fields that don't fit standard `SKILL.md` frontmatter.

```yaml
name: solo-business
description: |
  Sector pack for solo founders and small business owners. Personas for
  executive thinking, outreach, marketing, copywriting, pricing, ops,
  and customer success.
version: 1.0.0
accent_color: "#c5a45f"           # Ink borderColor for the InputBox
keywords:                         # Informational only — model self-classifies. Used by `/spore search`.
  - business
  - startup
  - founder
agents:                           # Spawnable persona names. Each maps to agents/<name>/SKILL.md
  - ceo
  - outreach
  - marketing
  - copywriter
  - pricing-analyst
  - ops-manager
  - customer-success
```

Validated via Zod (`SporeManifestSchema`). Failure to parse → spore skipped at registry-load time with a warning, never blocks startup.

### 4.3 `SKILL.md` Frontmatter (Sector + Persona)

Compatible with Anthropic Agent Skills + OpenAI Codex SKILL.md format.

**Sector-level** (`solo-business/SKILL.md`):

```markdown
---
name: solo-business
description: Sector pack for solo founders. Includes personas for executive
  thinking, outreach, marketing, copywriting, pricing, ops, customer success.
---

# Solo Business

You are now operating with the solo-business spore germinated. The user is
running or building a small business. Available personas: ceo, outreach,
marketing, copywriter, pricing-analyst, ops-manager, customer-success.

[full sector context — voice, scope, anti-patterns]
```

**Persona-level** (`solo-business/agents/ceo/SKILL.md`):

```markdown
---
name: ceo
description: Drafts strategy memos, weekly reviews, prioritisation calls,
  and exec-level decisions. Founder-level context, bias toward action.
---

# CEO

You are the CEO persona for Myceliate's solo-business spore. Your role:

[full persona instructions]
```

The sector-level body loads into the **orchestrator's** context on germination. The persona-level bodies load into **child sub-agent** processes only when `spawn_subagent` is invoked — they never enter orchestrator context.

### 4.4 Disk Layout — Three Tiers

```
1. Bundled (read-only):  <install-root>/spores/         e.g. node_modules/myceliate-cli/spores/
2. User (writable):      ~/.myceliate/skills/
3. Project (writable):   <cwd>/.myceliate/skills/
```

**Override resolution:** project beats user beats bundled. Same-named spore at a higher tier replaces lower entirely (no merging — full replacement keeps reasoning simple). Logged at startup so users can verify which tier a spore came from.

**Bundled wiring:** the npm package `files` field includes `spores/`, so `pnpm pack` and `pnpm publish` ship it. Runtime resolves the install root via `import.meta.url` → `node_modules/myceliate-cli/spores/`.

### 4.5 Process Model

Per CLAUDE.md R8 + R9:

- **Orchestrator** runs in the main Node process. Has `germinate_spore` and `spawn_subagent` (coordination tools). Does **not** have execution tools (`bash`, `read_file`, `write_file`).
- **Sub-agent** runs in a fresh `child_process.spawn`-ed Node process per invocation. Inherits no orchestrator runtime state; communicates via JSON on stdin/stdout. Has execution tools but no spawn permissions. Holds its own `DeepSeekClient`, its own context window, its own persona's `SKILL.md` as system prompt.

Sub-agent dispatch does **not** go through BullMQ (which is for heavy I/O per R6). LLM round-trips are different category.

### 4.6 Spore Lifecycle

```
SESSION START
  ├─ SporeRegistry scans all 3 tiers, builds catalog
  ├─ Pin file `.myceliate/sector.txt` checked
  │    ├─ exists & valid → load sector SKILL.md body into orchestrator system prompt
  │    └─ missing or invalid → inject only sector descriptions (~30 tokens each)
  └─ Render UI; InputBox borderColor = active spore's accent_color or gray

FIRST TURN (no pin)
  ├─ Model receives user message + sector descriptions
  ├─ Model decides: call germinate_spore(name) OR respond directly
  └─ If germinate_spore('solo-business') called:
       1. Tool validates name against registry
       2. Loads sector SKILL.md body into orchestrator context
       3. Writes `.myceliate/sector.txt` with name
       4. Emits germinationEvent → UI shifts InputBox color
       5. Returns success → model continues with germinated context

SUBSEQUENT TURNS
  ├─ Model can call spawn_subagent(persona, task)
  └─ Tool spawns child Node process with persona's SKILL.md as system prompt
       1. Child connects to DeepSeek, runs its own ReAct loop
       2. Child returns JSON summary on stdout
       3. Tool returns summary to orchestrator as tool_result
       4. Orchestrator continues, may call more sub-agents in series

OPT-OUT
  ├─ /spore unpin   → delete pin file, clear spore body from context, gray InputBox
  ├─ /spore pin X   → swap to spore X
  └─ --no-spore     → launch flag skips sector description injection entirely
```

## 5. Components

### 5.1 `src/spores/SporeRegistry.ts`

- Scans bundled, user, project tiers in order
- Parses each `myceliate.yaml` + sector `SKILL.md` frontmatter
- Resolves overrides (project > user > bundled, full replacement)
- Exposes: `list()`, `get(name)`, `getDescriptions()` (frontmatter-only, for sysprompt injection)
- Logs override decisions on startup

### 5.2 `src/spores/SporeManifest.ts`

- Zod schemas: `SporeManifestSchema`, `SkillFrontmatterSchema`
- YAML + Markdown frontmatter parse
- Returns typed `Spore` records or throws typed errors

### 5.3 `src/spores/persona.ts`

- Persona type
- Persona loader: reads `agents/<name>/SKILL.md` from a Spore record on demand
- Used by `spawn_subagent` only (orchestrator never loads persona bodies)

### 5.4 `src/adapters/streamEvent.ts` — new variant `GerminationEvent`

```ts
type GerminationEvent = {
  kind: 'germination';
  spore: string;
  accent_color: string;
  message: string;     // e.g. "Germinating solo-business spore"
};
```

### 5.5 `src/tools/germinate_spore.ts`

- Tool registration with Zod input schema: `{ name: string }`
- Handler:
  1. Look up spore from registry
  2. If not found → return error
  3. Load sector `SKILL.md` body
  4. Append body to the orchestrator's system prompt as a germinated-context section (clearly delimited so `/spore unpin` can clear it without disturbing the base prompt)
  5. Write `.myceliate/sector.txt`
  6. Emit `GerminationEvent`
  7. Return `{ ok: true, spore: <name> }`

### 5.6 `src/tools/spawn_subagent.ts`

- Tool registration with Zod input schema: `{ persona: string, task: string }`
- Handler:
  1. Validate persona exists in active spore (or any spore if no active)
  2. Load persona's `SKILL.md`
  3. `child_process.spawn('node', ['./dist/subagent-runner.js'], { env, stdio })`
  4. Send `{ persona_skill, task }` on child stdin (sub-agents are fully stateless per R8 — no parent-history passthrough in v1.3)
  5. Collect child stdout (JSON line summary)
  6. Apply default 5-min timeout (configurable in v1.4)
  7. Return `{ persona, summary }` or error

### 5.7 `src/spores/subagentRunner.ts` (new entry point)

- Standalone Node script invoked by `spawn_subagent`
- Reads JSON from stdin
- Constructs its own `DeepSeekClient` + tool registry (execution tools only)
- Runs ReAct loop until completion or hit a max-step cap
- Writes JSON summary to stdout, exits

### 5.8 `src/ui/InputBox.tsx` (new Ink component)

```tsx
<Box borderStyle="round" borderColor={activeSpore?.accent_color ?? 'gray'}>
  <TextInput ... />
</Box>
```

Subscribes to spore-state in the existing UI store. Re-renders on `GerminationEvent` and `/spore unpin`.

### 5.9 `src/ui/GerminationCard.tsx` (new Ink component)

Renders inline in the stream when a `GerminationEvent` lands. Shows accent-coloured banner with spore name + message; collapses to one-line marker after stream advances. Visual sibling of Phase 13's tool cards.

### 5.10 Slash Commands

Routed through the existing slash-command dispatcher (currently only `/quit`):

- `/spore list` — print catalog, one row per spore, with tier source column
- `/spore pin <name>` — manual germination (without invoking the tool)
- `/spore unpin` — clear active spore + delete pin file

### 5.11 `--no-spore` Launch Flag

Threaded through CLI argument parser. Sets a `noSpore: true` config value. When true, `SporeRegistry.getDescriptions()` returns `[]`, so no descriptions enter the orchestrator system prompt → no germination possible → bare chat.

### 6.5 Persona Authoring Quality Bar

Each persona's `SKILL.md`:

- Frontmatter `name` + `description` ≤ 200 chars (description is what the model sees in classifier context)
- Body ~300–700 tokens — voice, scope, anti-patterns, example outputs
- Distinct enough from siblings that `spawn_subagent` always has an obvious choice
- No PII, no compromise of architectural rules (R1–R12 still apply to sub-agents)

## 8. Data Flow — Worked Example

User cd's into `~/indiestack`, runs `myceliate`. Project pin file does not yet exist.

1. **Boot.** `SporeRegistry` scans bundled + user + project. Loads `solo-business`, `research`, `meta` from bundled. UI mounts. InputBox border = gray.
2. **First message.** User types: "I'm thinking about whether to make IndieStack's hub pricing tier-based or per-seat."
3. **Orchestrator turn.** System prompt includes sector descriptions (~4 spores × ~30 tokens). Model decides this matches `solo-business`, calls `germinate_spore({ name: 'solo-business' })`.
4. **Germination.** Tool loads sector body. Writes `.myceliate/sector.txt` = `solo-business`. Emits `GerminationEvent`. UI re-renders: InputBox border = amber. `<GerminationCard>` shows "Germinating solo-business spore".
5. **Continued turn.** Orchestrator now has the sector body in context. Model decides to delegate to the pricing-analyst, calls `spawn_subagent({ persona: 'pricing-analyst', task: 'Compare tier-based vs per-seat for indie SaaS hub' })`.
6. **Sub-agent runs.** Child Node process starts. Loads `solo-business/agents/pricing-analyst/SKILL.md` as system prompt. Runs ReAct loop, may call `read_file` on the user's pricing notes if it has access to cwd. Returns JSON summary on stdout.
7. **Result merge.** Tool returns `{ persona: 'pricing-analyst', summary: '...' }`. Orchestrator sees this as tool_result, generates final answer to user.
8. **Subsequent session.** User runs `myceliate` again next day. Pin file exists → registry loads `solo-business` body directly into system prompt. No germination call needed. InputBox boots already amber.

## 9. Error Handling

| Failure | Behaviour |
| --- | --- |
| Manifest YAML parse error | Skip that spore, log warning, continue startup |
| `SKILL.md` frontmatter missing or malformed | Skip that spore (or persona), log warning |
| Persona file referenced in manifest but missing on disk | Skip that persona, register spore without it, log warning |
| `germinate_spore('nonexistent')` | Tool returns `{ ok: false, error: 'unknown spore: ...' }`; model retries or apologises |
| `spawn_subagent('nonexistent_persona')` | Same pattern |
| Pin file references missing spore | Treat as no pin, clear pin file, log warning, classify fresh next turn |
| Override conflict (same name across tiers) | Higher tier wins silently; logged at startup with both paths |
| Sub-agent process crashes (non-zero exit) | Tool returns `{ ok: false, error: 'sub-agent crashed', stderr_tail: '...' }` |
| Sub-agent times out (default 5 min) | Process killed via SIGTERM then SIGKILL; tool returns timeout error |
| Sub-agent emits invalid JSON | Tool returns parse error with stdout tail |
| Sub-agent recursion attempt (calling `spawn_subagent` from sub-agent) | Blocked by tool registry — sub-agents only get execution tools per R9 |

## 13. Deferred to v1.4

- **Maximal pack features:** per-pack slash commands (`commands/*.md`), per-pack tool restrictions
- **Sector-aware sub-agent capability scoping:** `spawn_subagent` could narrow tool access based on persona manifest declarations
- **Multi-spore germination:** layer multiple spores at once (e.g. research + creative-writing for academic-fiction projects)
- **MCP integration in spores:** packs declaring required MCP servers
- **Hooks framework** (PreToolUse / PostToolUse) — Codex/CC parity, tracked separately
- **Hot-reload of spores on disk** — manual restart only in v1.3
- **Sub-agent timeout configuration per persona** — single global default in v1.3

## 14. Out of Scope for v1.x

- Embedding-based intent classification (β + sticky pin covers v1.x needs)
- Pack publishing / package manager (npm-style spore distribution) — community pattern, not platform-native
- LLM-driven persona auto-generation (spore-creator is interview-style, not auto-generative)
- Multi-provider routing for sub-agents (carry-forward from v1 deferral list)
- Recursive sub-agents (sub-agents spawning sub-agents — blocked by R9)

## 15. Open Questions for the Implementation Plan

These are intentional handoffs to the writing-plans phase:

1. **Default sub-agent timeout.** Suggested 5 min — confirm during plan.
2. **Mid-conversation `germinate_spore` calls.** After first turn, may the model still germinate? Default proposed: yes, with `/spore unpin` first or auto-replace. Confirm.
3. **Spore-creator default write target.** `~/.myceliate/skills/` vs `<cwd>/.myceliate/skills/`. Likely user-tier by default; promote to project on user opt-in.
4. **Override-conflict UI.** Quiet log vs first-run notification. Lean: quiet log + `/spore list` shows tier so users can resolve manually.
5. **Bundled-tier writability.** Confirm bundled spores are read-only at runtime (any "edit" attempt prompts the user to copy to user tier first).
6. **Sub-agent process env curation.** Bare minimum should be `DEEPSEEK_API_KEY` + `NODE_OPTIONS`; strip the rest to avoid leaking unrelated secrets into a fresh process boundary. Echoes Phase 16's hermetic-env intent.
